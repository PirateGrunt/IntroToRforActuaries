    • Overview

    What will you do in this workshop? What won’t you do? You’ll hear all about how tremendous R is, where it came from, where it’s going and how it has become the tool of choice for tens of thousands of analysts across the world. You’ll learn how to make sure it runs smoothly despite the objections of your company’s IT department. You’ll learn how to exploit the knowledge of other R users. You’ll learn how to import and export information from text files, Excel files, databases and the web. You’ll draw pictures of the probability density function for loss development factors. You’ll use monte carlo simulation. You’ll learn how to use generalized linear modeling to predict the results of football games. You’ll create a color-coded map to show which states produce poor underwriting results. What else? If time permits, you might use cluster analysis. Or, you might build a word cloud to mine text information from your claims data.

    What is R?

    R is a programming language, derived from S. It is an "open source" project, which means that its license entitles anyone to use the software, so long as they keep their use of the software "open". The specific licensing language may be found here: http://www.r-project.org/COPYING. The practical consequence of an open source project is that- similar to something like Wikipedia- users may contribute improvements and extensions to the project.

    Information about R may be found at http://www.r-project.org/. Another great source for inspiration is http://www.r-bloggers.com/. Here, you'll see examples where R has been applied in dozens of different contexts. This is an aggregation of blogs from around the world and changes every day. It's always worth checking out. There are at least two R-focused blogs written by actuaries. The first is http://lamages.blogspot.com/ written by Markus Gesman in London. Markus works for Lloyds and is the original author of the ChainLadder package in R. Another, http://pirategrunt.com/, is written by me. This is where I write about stuff that interests me in a format where I needn't worry quite so much about how polished the verbiage and analysis is. I'll give the standard caveat that the views expressed on my blog do not necessarily reflect the views of my employer, the CAS or anyone other than me.

    Installing R

    Participants will get the most out of the session if they show up with a laptop which has R and R Studio already installed. This will enable folks to follow along with the examples and ask for help in real time. The first place to look for installation is http://cran.r-project.org/mirrors.html

    Installing R Studio

    Installing R is most of the battle. Depending on the sort of person you are, it may even be all of the battle. R comes with a fairly spartan user interface, which is sufficient

    It all sounds so easy, but in practice this is rarely the case. One of the difficulties lies in navigating the infrastructure developed by your company's IT department. I've written some suggestions on how to deal with your IT department, which I hope are useful.
